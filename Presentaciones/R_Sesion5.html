Mannejo de datos con dplyr

El paquete dplyr es uno de los paquetes m√°s √∫tiles para manejar datos. Permite manipular de manera m√°s intuitiva los datos para exploraci√≥n y graficaci√≥n.

Este paquete forma parte de un conjunto de paquetes dise√±ado para facilitar todo el proceso de an√°lisis interactivo. Este conjunto de paquetes se llama tidyverse y entre los m√°s importantes que lo conforman est√°n los siguientes:

  • dplyr: herramientas para la manipulaci√≥n de datos.
  • ggplot2: generaci√≥n de gr√°ficas basadas en la gram√°tica de gr√°ficas de Leland Dikinson.
  • readr: herramientas para importaci√≥n de datos.
  • tidyr: paquete para la limpieza de datos

Y hay muchos otros.

Estos paquetes utilizan una nueva estructura de datos, que m√°s bien es una estructura ‚Äòmejorada‚Äô, llamada tibble que es como un data.frame con esteroides. Tambi√©n utilizan la idea de composici√≥n entre funciones para encadenar acciones \(f \circ g (x) = f(g(x))\), a trav√©s del operador magritte: %>%.

dplyr introduce las siguientes acciones, funciones o verbos: - select(): selecciona variables (columnas) por nombre - filter(): selecciona observaciones (renglones) por valor - arrange(): reordena observaciones - mutate(): transforma variables a partir de las existentes - summarize(): colapsa variables a trav√©s de funciones sumarias

Estas acciones se combinan con la funci√≥n group_by para considerar operaciones por grupos.

Leer detos de archivos con delimitadores raros

T√≠picamente la mejor manera de trabajar con archivos de Excel que tienen una estructura de dataframe es guardarlos como archivos de texto separados por comas (csv) y leerlos con la funci√≥n read.csv. Si el delimitador no es una coma, se puede usar read.delim. Pero en nuestro caso estas opciones no sirven.

Los archivos usan como delimitador dobles dos puntos, ‚Äú::‚Äù y R no puede separar f√°cilmente estos datos. Por lo tanto, tenemos que buscar la manera de poder leer el archivo y generar un dataframe para poder comenzar el an√°lisis.

La estrategia es primero leer los archivos por lineas, despu√©s romper las l√≠neas en donde est√°n los delimitadores, y despu√©s armar el dataframe. Usaremos readLines para este prop√≥sito

pelis <- readLines("https://raw.githubusercontent.com/jvega68/EA3/master/datos/movies.dat.txt")

califs <- readLines("https://raw.githubusercontent.com/jvega68/EA3/master/datos/ratings.dat.txt")

usuarios <- readLines("https://raw.githubusercontent.com/jvega68/EA3/master/datos/users.dat.txt")

Rompemos las lineas en los lugares donde haya ‚Äú::‚Äù. Cada linea se convierte en la entrada de una lista:

pelis <- strsplit(x = pelis, fixed = T, split = "::", useBytes = T)
califs <- strsplit(x = califs, fixed = T, split = ":", useBytes = T)
usuarios <- strsplit(usuarios, "::", fixed = TRUE, useBytes = T)

Aqu√≠ usamos el par√°metro useBytes = T porque la codificaci√≥n del archivo puede hacer que perdamos informaci√≥n, ya que no sabemos c√≥mo est√°n codificados los datos originales.

Ahora ya podemos convertir a un dataframe

pelis <- data.frame(matrix(unlist(pelis), ncol = 3, byrow = T), stringsAsFactors = F)
califs <- data.frame(matrix(unlist(califs), ncol = 4, byrow = T), stringsAsFactors = F)
usuarios <- data.frame(matrix(unlist(usuarios), ncol = 5, byrow = T), stringsAsFactors = F)

Noten que estos dataframes no tienen los nombres de los campos y todas las columnas son de tipo caracter, por lo que los n√∫meros no son n√∫meros. Tenemos que arreglar todos estos detalles.

EDA

EDA significa Exploratory Data Analysis, es un t√©rmino acu√±ado por John Tukey para referirse al an√°lisis estad√≠stico inicial al que se deben someter los datos que se obtienen para poder identificar ptrones y caracter√≠sticas generales de los datos, tendencias, caracter√≠sticas num√©ricas, etc. y tener un mayor entendimiento y conocimiento inicial para formular preguntas correctas.

Esta fase tambi√©n se conoce actualmente como an√°lisis estad√≠stico descriptivo de los datos y tiene las siguientes finalidades:

  • Maximizar la visi√≥n sobre el conjunto de datos.
  • descubrir estructuras de datos subyacentes
  • identificar las variables que son m√°s relevantes
  • detectar anomal√≠as y valores at√≠picos
  • probar los supuestos subyacentes de los datos
  • desarrollar modelos adecuados.

La mejor manera de llevar a cabo un an√°lisis exploratorio es hacerlo de manera interactiva, junto con el paquete que usamos, e ir anotando las caracter√≠sticas m√°s relevantes de la informaci√≥n, pero tambi√©n las acciones que llevamos a cabo para que podamos reproducir nuestro an√°lisis.
